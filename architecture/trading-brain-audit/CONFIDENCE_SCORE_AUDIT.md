# Confidence Score Audit

## Equity Confidence

For OpenAI-assisted equities:

- Confidence is generated by OpenAI in `OpenAIProposalAnalyzer.propose`.
- Prompt requires `confidence_score >= GuardrailConfig.min_confidence_score`.
- There is no deterministic formula for OpenAI equity confidence in code.
- The model is not shown historical outcomes or calibration tables.

For demo equities:

- Confidence = `max(guardrails.min_confidence_score, 0.72)`.
- This is validation-only and not statistically meaningful.

## Crypto Confidence

For Kraken crypto:

- `propose_crypto_trades` sets `confidence = row["overall_due_diligence_score"]`.
- It requires confidence >= `settings.auto_trade.min_confidence`.
- It also requires `technical_trend_score > 0.5`.

Crypto score creation:

- `record_crypto_research_score` accepts supplied metrics.
- If `overall_due_diligence_score` is missing, it computes:

```text
overall = (technical + momentum + risk + sentiment + liquidity) / 5
```

where missing component values are converted to 0.0 by `safe_score(...) or 0.0`.

CoinGecko-derived fields:

```text
technical_trend_score = clamp(0.5 + 7d_percent_change / 100)
momentum_score = clamp(0.5 + 24h_percent_change / 100)
volatility = min(1.0, abs(30d_percent_change) / 100)
liquidity = min(1.0, volume_24h / market_cap)
risk_score = 1.0 - volatility
```

Approved-pair fallback:

- `_bootstrap_crypto_universe_from_kraken_permissions` seeds heuristic values.
- Overall/confidence = `max(auto_trade.min_confidence, 0.85)`.

## Investment Score Confidence

`calculate_investment_score` computes:

```text
fundamental = proposal_confidence if news_summary else 0
technical = safe_score(technical_summary) or proposal_confidence if technical_summary else 0
market = safe_score(market_sentiment_summary) or proposal_confidence if market_sentiment_summary else 0
macro = proposal_confidence if macro context exists else 0
behavioural = proposal_confidence if behavioural context exists else 0
policy = philosophy_fit
risk = clamp(1 - stop_loss_pct)
overall_confidence = average(fundamental, technical, market, macro, behavioural, policy, risk)
```

This is a heuristic average. Text fields such as `"Positive trend."` may map through `safe_score` or fall back to proposal confidence.

## Thresholds

Key thresholds:

- `GuardrailConfig.min_confidence_score`: default 0.65.
- `AutoTradeConfig.min_confidence`: default 0.85.
- `AutoTradeConfig.min_philosophy_fit`: default 0.85.
- Recommendation freshness:
  - >= 0.85: 4-hour lifetime.
  - >= 0.75: 12-hour lifetime.
  - otherwise: 24-hour lifetime.

## Missing-Data Treatment

Current treatment is inconsistent by layer:

- Crypto score aggregation turns missing components into 0.0.
- Due diligence marks macro/behavioural as insufficient when no evidence exists.
- Investment scoring assigns 0 to missing macro/behavioural evidence.
- OpenAI equity confidence can be returned without calibrated missing-data penalty.
- Recommendation display uses fallback score payload when orchestrator score is missing.

## Audit Answers

Does 85% confidence have a mathematically defensible meaning?

- No. It is a heuristic threshold, not a calibrated probability.

Is it calibrated against historical outcomes?

- No. No calibration model found.

Is it comparable between equities and crypto?

- No. Equity confidence can be model-generated; crypto confidence is deterministic heuristic score.

Is it comparable between different strategies?

- No. Formal strategies are not identified, and confidence is not strategy-normalized.

Is it primarily a heuristic score?

- Yes.
